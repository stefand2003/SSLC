Optimisation

During the development stage of the website, especially for a heavy page like "listing all the members" I would have considered webP as it is a modern web image format that creates small images - compacts them so they take up less storage and thus making the web faster. However webP turned out to be slower than jpeg.

I made sure to use the Developer tools to measure image loading times - I even switched to incognito mode to prevent cached images and obtain accurate loading time results.

I found out that the loading time for jpeg images is 824ms which is < 1second. Where as If I was using a webP it came down to 2.7s which is nearly 3 and a half times slower than the use of JPEG. That is why I will be using jpeg instead of webp.

How Wave helped me?

Wave is a good developer tool because it comes as an extension and can work with you while you are coding. It identified, flagged and displayed my error for me to see and take notice of in order to improve web page accessibility.

In my Membership page, I initially had the same alt text, "student working in a cafe", for all the images were the same. They all depicted a student working in a cafe. However with wave's help, the realisation of that prompted me to assign them all different alt texts.
